## 👋 Welcome to CloudDeployLab

**CloudDeployLab** is an open organization built around one simple idea — **sharing practical DevOps knowledge through real, working projects.**

Every repository here is something you can clone, run, study, and adapt. No slides, no theory-only content. Just hands-on configurations, pipelines, Helm charts, playbooks, and deployment setups that reflect how things actually work in the field.

Whether you are just starting your DevOps journey or looking to sharpen specific skills around Kubernetes, CI/CD, or infrastructure automation — this is a place to **learn by doing**, and to grow alongside others doing the same.

---

## 🎯 Why This Org Exists

DevOps is a broad, fast-moving field. It can be hard to know where to start, what to focus on, or how to connect the pieces together. CloudDeployLab was created to address exactly that — by providing:

- **Structured, real-world examples** across the full DevOps lifecycle
- **Reusable templates** for CI/CD, Kubernetes, Helm, and infrastructure
- **A learning path** from beginner setups to production-grade deployments
- **A sharing culture** where knowledge is open, documented, and accessible to everyone
- **Exposure to modern tools** used in real engineering teams today

---

## What You'll Find Here

Our repositories are organized around the core pillars of modern DevOps:

### CI/CD Pipelines
End-to-end pipeline examples for building, testing, scanning, and deploying applications — covering both **fullstack** and **microservice** architectures.

- Jenkins pipelines with stages for code scanning, Docker image build & push, and deployment
- GitHub Actions workflows for automated testing and delivery
- CI/CD patterns for Spring Boot, React, Node.js, and more

### Kubernetes & Helm
Practical guides and configurations for deploying applications on Kubernetes — from a lightweight **K3s** local cluster to a full **production K8s** setup.

- Deploy Spring Boot microservices to Kubernetes
- Helm charts for microservices, fullstack apps, and monolithic applications
- ConfigMaps, Secrets, Ingress, HPA, and other K8s fundamentals in context

### Infrastructure Automation
Ansible playbooks and automation scripts to provision and configure infrastructure repeatably and reliably.

- Automate full K8s and K3s cluster setup from scratch
- Server provisioning playbooks for common DevOps tooling
- Infrastructure-as-Code practices applied to real scenarios

## Who Is This For?

CloudDeployLab is for anyone who wants to **learn DevOps through practice** — not just read about it.

| Audience | How This Helps |
|---|---|
| **Developers** | Understand how your code gets built, containerized, and deployed |
| **DevOps Engineers** | Find reusable pipelines, Helm charts, and automation playbooks |
| **Students & Learners** | Build a portfolio of real projects using industry-standard tools |
| **Contributors** | Share your knowledge and help improve resources for others |

No gatekeeping. No prerequisites. Start wherever makes sense for you, and go at your own pace.

---

## Contributing

CloudDeployLab grows through community contributions. If you have a pipeline pattern, Helm chart, playbook, or deployment approach worth sharing — this is the place for it.

**To contribute:**

1. Fork the repository you want to improve
2. Create a feature branch — `git checkout -b feat/your-topic`
3. Make your changes with clear, documented code
4. Open a Pull Request with a description of what you added and why
5. We'll review and merge it together

Have an idea for a new topic or tool? Open an **Issue** or start a **Discussion** — suggestions are always welcome.
